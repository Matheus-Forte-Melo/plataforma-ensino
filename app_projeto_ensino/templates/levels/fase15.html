Operadores Relacionais e Lógicos

Os operadores relacionais e lógicos são ferramentas essenciais para comparar valores e avaliar expressões. Eles permitem realizar verificações e tomar decisões com base em condições.

Operadores Relacionais

Os operadores relacionais são usados para comparar dois valores e retornam um resultado booleano: verdadeiro (true) ou falso (false).

Operador Significado Exemplo Resultado
> Maior que 5 > 3 true
< Menor que 2 < 7 true
>= Maior ou igual 4 >= 4 true
<= Menor ou igual 3 <= 2 false
== Igual a 10 == 10 true
!= Diferente de 8 != 5 true

Operadores Lógicos

Os operadores lógicos combinam expressões booleanas (ou condições) e retornam um resultado booleano. Eles são úteis para verificar múltiplas condições ao mesmo tempo.

Operador Nome Descrição Exemplo Resultado
&& AND Retorna true apenas se todas as condições forem true (5 > 3) && (2 < 7) true
|| OR Retorna true se pelo menos uma condição for true (5 > 3) || (2 > 7) true
! NOT Inverte o valor lógico: true vira false e vice-versa !(5 > 3) false

Exemplos Didáticos

  1. Comparação simples (Operadores Relacionais):

    Imagine que você quer verificar se um número A é maior que um número B.

    • Se A = 10 e B = 5, então: A > B → 10 > 5 resulta em true.
    • Se A = 3 e B = 7, então: A < B → 3 < 7 resulta em true.

  2. Combinação de condições (Operadores Lógicos):

    Suponha que você quer validar duas condições ao mesmo tempo:

    • Um número é maior que 0 e menor que 10.
    • Para X = 5: (X > 0) && (X < 10) → (5 > 0) && (5 < 10) resulta em true.
    • Para X = 15: (X > 0) && (X < 10) → (15 > 0) && (15 < 10) resulta em false.

  3. Negação de uma condição (NOT):

    Se quisermos verificar se um número não é igual a 10:

    • Para Y = 10: !(Y == 10) → !(true) resulta em false.
    • Para Y = 5: !(Y == 10) → !(false) resulta em true.

Como esses operadores se conectam

Os operadores relacionais criam condições básicas que retornam valores booleanos. Esses valores booleanos podem ser combinados com os operadores lógicos para construir condições mais complexas.

Exemplo:

(A > B) && (C != D): Aqui, estamos verificando duas condições:

  1. Se A é maior que B.
  2. Se C é diferente de D.

O resultado final será true apenas se ambas as condições forem verdadeiras.
